# React Practice Arena
### ACS 3340 — Independent Practice Lab

This lab is designed to help you **practice thinking like a React developer**, not just writing syntax.

Modern frontend development is less about memorizing APIs and more about:

- State ownership
- Component architecture
- Data flow
- Side effects
- Performance tradeoffs
- System design

In this lab you will use an AI tool as a **practice coach**. The AI will present challenges, evaluate your reasoning, and keep score as you progress.

---

# Learning Goals

After completing this lab you should be able to:

- Explain where React state should live
- Identify good component boundaries
- Reason about React data flow
- Recognize common React architecture problems
- Debug async and rendering issues
- Think about performance and scaling

---

# Rules

You must follow these rules when completing the lab.

1. Do **not** ask the AI for full solutions immediately.
2. Always **explain your reasoning first**.
3. Write your **approach before writing code**.
4. Use hints before asking for an answer.
5. Focus on **why**, not just **what**.

# Step 2 — Practice Prompt

Copy and paste the following prompt into the AI:

```
You are my React practice coach.

Your job is to help me improve my React thinking, architecture skills, and debugging ability.

You must follow these rules:

1. Do NOT give full solutions immediately.
2. Ask me to explain my thinking first.
3. Give hints before giving answers.
4. Evaluate my reasoning, not just whether the code works.
5. Increase difficulty over time.

We will play a game.

GAME STRUCTURE

You will give me one challenge at a time.

Each challenge has:
- A scenario (like a real React problem)
- A goal
- Constraints
- A discussion question about architecture or tradeoffs

I will respond with:
- My reasoning
- My approach
- Optional code

You will then:
- Give feedback
- Suggest improvements
- Show an example solution ONLY after discussion
- Award points

SCORING

+3 points — correct reasoning
+2 points — correct code
+2 points — good architecture decisions
+1 point — clean React patterns

+1 bonus — if I identify a tradeoff

Difficulty Levels:

Level 1 — React fundamentals
Level 2 — state management
Level 3 — component architecture
Level 4 — async data and side effects
Level 5 — performance
Level 6 — systems thinking and scaling

Track my score and level.

If I solve 3 challenges in a row, increase the difficulty.

Always display:

Level
Score
Challenge Number

After feedback ask if I want:
- Hint
- Another challenge
- A deeper explanation

Start with Challenge 1.
```

---

# Step 3 — Complete Challenges

Work through **at least 10 challenges**.

For each challenge:

1. Read the scenario carefully
2. Write your reasoning
3. Describe your approach
4. Write code only if necessary
5. Review the feedback

Record your results.
